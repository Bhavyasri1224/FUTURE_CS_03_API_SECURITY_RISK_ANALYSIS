# API Security Risk Analysis

## Objective
Perform a read-only API security assessment on a public API and identify common security risks.

## API Tested
JSONPlaceholder Test API

## Tools Used
- Postman
- Browser Developer Tools
- MS Word
- GitHub

## Scope
- Documentation review
- Read-only GET requests
- Header and response inspection

## Methodology
1. Selected JSONPlaceholder API.
2. Tested endpoints using Postman.
3. Analyzed responses.
4. Identified security risks.
5. Classified severity.
6. Suggested remediation.

## Risks Identified
- Missing Authentication
- Excessive Data Exposure
- Missing Rate Limiting
- Input Validation Issues

## Conclusion
This assessment demonstrates how common API security risks can affect SaaS applications and highlights best practices for securing APIs.
